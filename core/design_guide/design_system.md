# design system

Collabit のデザイン基準をまとめるファイルです。

このファイルでは、次の 2 種類を分けて扱います。

- ブランドのデザインシステム
  - 案件全体の見せ方、信頼感、情報整理の考え方
- 記事アセットのデザインシステム
  - 記事ごとのサムネイルや説明用画像の作り方

## 1. Brand System

### Brand Direction

- 信頼感がある
- わかりやすい
- 誇張しすぎない
- 伝えたい価値が視覚的に整理されている

### Visual Rules

- 情報の優先順位を先に決める
- 1 画面 1 メッセージを意識する
- 装飾より可読性を優先する
- テキスト量が多い場合は余白で整理する

### Layout Notes

- 情報量が多い場合も、見出し、余白、強弱で読みやすさを作る
- 伝えたい価値が複数ある場合は、重要度順に並べる
- 強い訴求を使う場合も、誇張より整理を優先する

## 2. Article Asset System

記事フォルダ内の `assets/` では、次の 2 種類を扱います。

- `thumbnail/`
  - サムネイル、アイキャッチ、SNS 用カバー画像
- `description/`
  - 記事説明用画像、本文図版、補助ビジュアル

### Thumbnail Rules

- 一目でテーマが伝わることを優先する
- 文字を載せる場合は、短く、読みやすく、情報を詰め込みすぎない
- サムネイル単体で過剰に煽らない
- 記事本文のトーンとずれない見せ方にする

### Description Asset Rules

- 記事本文の理解を助ける画像にする
- 装飾目的だけの画像を増やしすぎない
- 図版は、文章より早く意味が伝わることを重視する
- 本文の論点と一致する内容にする
- 記事本文とクリエイティブのトーンをそろえる

## 3. Asset Handling

- 元データと書き出しデータを区別する
- 命名は日付か用途を含めて判別しやすくする
- 軽量な補助素材はローカルの `assets/` に置く
- 重い素材や共有前提データは必要に応じて外部ストレージも使う

## 4. Review Checklist

- 目的に合った見せ方になっているか
- 誤字、表記ゆれ、視認性の問題がないか
- 納品形式や書き出しサイズが合っているか
- トンマナが案件文脈とずれていないか
- ブランド全体の見せ方と、記事単体の目的が両立しているか
