# brief

## Basic Info

- 記事タイトル案: 売却か賃貸か、結局どっちが得？自宅・投資用不動産を3回以上売買した不動産鑑定士にインタビューしてこの問題に最終決着！
- 目的: 売却と賃貸それぞれの「お金＋手間＋リスク」を1枚の表とシミュレーションでイメージできるようにし、自分に合う判断軸を持ってもらう。→ 一括査定時には賃貸と売却の2つの査定を同時に依頼できる、という方向にもっていく
- 想定読者: 自宅・実家・相続物件を売るか貸すか迷っている人。転勤・相続・住み替えなどライフステージの変化が契機
- 検索意図: 「売るか貸すか どっちが得」「売却 賃貸 比較 シミュレーション」「転勤 自宅 売るか貸すか」など
- CTA: AI査定で「売るといくら」「貸すといくら」をまず確認 → 不動産会社に売却査定＋賃貸査定を同時依頼
- 納期: 未記載

## 構成レビュー（要確認・要方針決定）

**① リードと本文の矛盾【要修正】**
- リードの「本記事でわかること」に「お金・手間・リスクを1枚の表で比較」と記載があるが、比較表は削る方針のため記述が矛盾している
- → リードの該当箇所を修正する

**② タイトルと記事結論のトーンのズレ【要方針確認】**
- タイトルは「最終決着！」と断言しているが、記事の結論は「物件・状況・ライフプランによって変わる」でまとめている
- 「最終決着」＝「ケースバイケース」は成立しない
- → タイトルのトーンを弱めるか、本文でより明快に「このケースは売る／このケースは貸す」と結論を出すか、方針を決める必要がある

**③ インタビューの位置がタイトルと乖離している【要方針確認】**
- タイトルで「不動産鑑定士にインタビューして最終決着」と打ち出しているのに、インタビューは3番目のH2（記事中盤）にある
- タイトルでインタビューを前面に出すなら、記事の柱として最初か最後に置くのが自然
- 現状は補強コンテンツ扱いになっており、タイトルとの位置づけが合っていない
- → 「インタビューを記事の核に据える」か「タイトルからインタビューの要素を外す」か方針を決める必要がある

**④ フローチャートとチェックリストの役割が重複【要整理】**
- どちらも「あなたは売却向き？賃貸向き？」を判断させる構成になっている
- 差別化の案：フローチャート＝物件・収支の客観的条件で絞り込む／チェックリスト＝性格・希望などの主観的条件で確認する
- → 役割を明確に差別化するか、どちらかを削るか検討が必要

**⑤ シミュレーションとケーススタディが重複【削除推奨】**
- 「今売る vs 10年賃貸」のシミュレーションの直後にケーススタディが続いており役割が重複している
- ディレクター自身も「無しでもいいかも」と言及済み
- → ケーススタディは削除が現実的。シミュレーションで完結させる

**⑥ よくある疑問の位置が遅い【要検討】**
- 現在まとめの直前に配置されているが、読者の疑問（「転勤の場合は？」「同時に進められる？」）はメリデメを読んだ直後かフローチャートの後に解消されるべき
- → フローチャートの後あたりへの移動を検討

**⑦ インタビューQが9問は多い【要絞り込み】**
- Q5（売って後悔した一言）とQ6（貸しておいてよかったケース）は内容が近く重複感あり
- 読者体験として4〜5問が上限
- → 取材前に質問を絞り込む必要がある
